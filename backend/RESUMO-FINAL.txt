🐛 CRÍTICOS (3 corrigidos)

1. ✅ SINTAXE POSTGRESQL - routes/usuarios.js
   • Problema: Usava sintaxe MySQL em projeto PostgreSQL
   • Solução: Convertido para PostgreSQL ($1, $2, result.rows)
   • Impacto: 3 rotas corrigidas

2. ✅ CANCELAMENTO DE PEDIDOS - controllers/pedidoController.js
   • Problema: Ordem de testes causava status 'enviado' antes de cancelar
   • Solução: Reordenados testes + invertida lógica para whitelist
   • Impacto: 1 rota corrigida

3. ✅ SCHEMA DE CUPONS - controllers/cupomController.js
   • Problema: Campos não correspondiam ao banco de dados
   • Solução: Corrigidos 4 campos (valor_minimo_pedido, descricao, etc)
   • Impacto: 3 rotas corrigidas


⚠️ MÉDIOS (3 corrigidos)
─────────────────────────────────────────────────────────────────────────────

4. ✅ VALIDAÇÃO DE SENHA - controllers/authController.js
   • Problema: Só aceitava 'senhaNova'
   • Solução: Aceita 'senhaNova' ou 'novaSenha'

5. ✅ ATUALIZAÇÃO DE ESTOQUE - controllers/produtoController.js
   • Problema: Só aceitava campo 'estoque'
   • Solução: Aceita 'estoque' ou 'quantidade'

6. ✅ NEWSLETTER CANCELAMENTO - controllers/newsletterController.js
   • Problema: Campo data_atualizacao não existe no banco
   • Solução: Removidas referências + adicionada rota POST


ℹ️ MENORES (2 melhorias)
─────────────────────────────────────────────────────────────────────────────

7. ✅ FLEXIBILIDADE EMAIL - newsletterController.js
   • Aceita email em body ou query params

8. ✅ MENSAGENS DE ERRO - pedidoController.js
   • Incluído status atual nas mensagens
